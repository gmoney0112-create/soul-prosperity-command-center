    GHL Build Sheet

    Translate the funnel into a GoHighLevel build

    This is the operator build sheet. Open GHL in one tab, this page in another, and walk it top to bottom: tag taxonomy, custom fields, pipeline, workflow-by-workflow implementation, then QA before traffic. Everything here is static — no data is saved in the browser.

    Step 1

    Tag taxonomy

    Build these tags in Settings → Tags first. Every workflow below reads or writes at least one of them. Use lowercase, dash-separated names so filters and exports stay predictable.

    Step 2

    Custom fields

    Create in Settings → Custom Fields → Contact. Keep the field keys stable — workflows, email templates, and Skool invite links all reference them directly.

    Field Type Purpose
    first_purchase_dateDateSort buyers for testimonial ask timing
    last_purchase_productTextMost recent paid offer key (ebook7, audio17…)
    ltv_centsNumberLifetime value in cents, incremented per purchase
    freebie_sourceTextWhich post or channel drove the opt-in
    skool_invite_sentDatePrevents double-inviting on re-entry
    refund_reasonTextCaptured when refund is requested, used by WF-08
    nps_scoreNumberCaptured post-course for testimonial routing
    preferred_channelDropdownemail · sms · both — respected by all sends

    Step 4

    Workflow-by-workflow build

    Build each of the eight workflows in Automation → Workflows. Names are exact — copy from the block below so filter-by-name stays clean. Every workflow ends in Wait until exit condition so a later purchase immediately stops the current pitch.

    Paste-ready workflow names
    WF-01

    Freebie nurture

    Trigger
    Form submitted: Freebie opt-in
    Entry action
    Add tag lead-freebie; set freebie_source; send delivery email with download + $7 pitch.
    Sequence
    Day 0 deliver · Day 1 story email · Day 2 SMS nudge (respect preferred_channel) · Day 3 $7 pitch email · Day 5 final pitch.
    If / else
    If opened Day 3 but no click → send Day 4 objection email. Else skip to Day 5.
    Exit condition
    Tag buyer-ebook7 applied OR unsubscribed-email applied.
    WF-02

    $7 buyer delivery

    Trigger
    Order submitted / form payment: $7 eBook product.
    Entry action
    Add buyer-ebook7; remove lead-freebie is optional (keep for source); set first_purchase_date, last_purchase_product=ebook7; increment ltv_cents by 700.
    Sequence
    Minute 0 delivery email with PDF · Minute 5 SMS "check your inbox" · Hour 1 $17 audio pitch · Day 1 $27 paperback pitch · Day 2 story.
    If / else
    If buyer-audio17 present → skip audio pitch. If buyer-paperback27 → skip paperback pitch.
    Exit condition
    Tag buyer-course67 (jump to WF-05) OR refunded.
    WF-03

    Order bump recovery ($17 audio)

    Trigger
    Tag added: buyer-ebook7.
    Wait
    2 hours, then check for buyer-audio17. If present, exit.
    Sequence
    Hour 2 audio bridge email · Day 1 SMS "still reading? listen instead" · Day 3 final $17 offer.
    If / else
    If preferred_channel=email, skip SMS steps.
    Exit condition
    buyer-audio17, buyer-course67, or refunded.
    WF-04

    Paperback upgrade ($27)

    Trigger
    Tag added: buyer-ebook7 OR buyer-audio17.
    Wait
    Day 2 after trigger.
    Sequence
    Day 2 study-copy pitch · Day 4 gift-copy angle · Day 7 last call with scarcity copy.
    If / else
    If buyer-paperback27 present → exit immediately.
    Exit condition
    buyer-paperback27, buyer-course67, or refunded.
    WF-05

    $67 course pitch

    Trigger
    Tag added: buyer-ebook7, or engagement score high (multi-open, multi-click).
    Wait
    Day 5 after first purchase.
    Sequence
    Day 5 method teaser · Day 6 case study · Day 7 curriculum email · Day 8 SMS CTA · Day 10 final deadline email.
    If / else
    If buyer-paperback27 → lead with "you already have the book, now build yours." Else default method copy.
    Exit condition
    buyer-course67, lifetime, or refunded.
    WF-06

    Skool 7-day trial invite

    Trigger
    Tag added: buyer-course67 OR engagement threshold.
    Entry action
    Set skool_invite_sent to today.
    Sequence
    Day 0 trial invite email with Skool link · Day 2 SMS check-in · Day 5 social proof · Day 6 "trial ends tomorrow".
    If / else
    If skool-trial already present → exit.
    Exit condition
    skool-trial, skool-monthly, skool-annual, or lifetime.
    WF-07

    Annual / lifetime upgrade

    Trigger
    Tag added: skool-monthly (day 21), or skool-annual (day 60).
    Sequence
    For monthly → pitch $247/yr annual. For annual → pitch $497 lifetime at day 60 + 30-day save path.
    If / else
    Branch on current sub tag to pick the next-rung pitch.
    Exit condition
    lifetime applied, or sub cancelled (moves to WF-08 save).
    WF-08

    Abandoned checkout & refund save

    Trigger A
    Checkout started, not completed in 30 min → add abandoned-checkout.
    Trigger B
    Refund requested / sub cancel → add refund-risk; capture refund_reason.
    Sequence A
    30 min email "finish your order" · Hour 4 SMS · Day 1 bonus-added email · Day 2 final.
    Sequence B
    Minute 5 human-tone email from Terrance · Hour 2 offer downgrade / extension · Day 1 pause-instead-of-cancel option.
    Exit condition
    Matching buyer tag applied (A) OR refunded finalized (B).
